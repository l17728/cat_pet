## 1. 🏥 健康与福祉系统

### 核心理念

猫咪的健康不再由单一指标决定，而是由**五维福祉**综合影响：

| 维度 | 权重 | 说明 |
|------|------|------|
| 🍖 饮食 | 20% | 饱食度，影响基础生存 |
| ⚡ 睡眠 | 20% | 精力值，影响活动能力 |
| 💖 心情 | 20% | 心理健康，影响性格表现 |
| 💧 水分 | 15% | 水平衡，新增独立指标 |
| 👥 社交 | 10% | 社交需求，影响幸福感 |
| 🛁 清洁 | 15% | 卫生状况，影响疾病风险 |

### 健康评分

- **90-100**: 💪 非常健康 - 性格特质更加明显，互动意愿高
- **75-89**: 😊 健康 - 正常表现
- **60-74**: 😐 一般 - 有些无精打采
- **40-59**: 😟 亚健康 - 容易烦躁，不愿互动
- **0-39**: 🚨 危险 - 需要立即关注！

### 性格影响

健康度会影响性格表现：

| 性格 | 健康时 | 亚健康时 |
|------|--------|----------|
| 活泼 | 上蹿下跳 | 变得异常安静 |
| 温顺 | 乖乖配合 | 更加依赖主人 |
| 高冷 | 优雅独立 | 完全不理人 |
| 粘人 | 撒娇可爱 | 不停叫唤求关注 |
| 独立 | 自主玩耍 | 拒绝任何互动 |

### 命令

```bash
健康检查      # 查看详细健康报告
状态详情      # 查看五维福祉状态
```

## 2. 🌀 时空门系统

### 功能说明

打开时空门可以生成**随机的魔幻空间**，猫咪可以在其中探索玩耍。

### 场景稀有度

| 稀有度 | 概率 | 示例场景 |
|--------|------|----------|
| ⚪ 普通 | 50% | 神秘森林、金色海滩 |
| 🟢 少见 | 20% | 云端山峰、金色沙漠 |
| 🔵 稀有 | 15% | 水晶洞穴 |
| 🟣 史诗 | 10% | 浮空岛屿、星空花园 |
| 🟡 传说 | 4% | 时光遗迹 |
| 🎁 季节 | 1% | 樱花秘境、冬日仙境 |

### 存储限制

- 最多存储 **5 个** 魔幻空间
- 每个空间可以重复访问
- 每次访问可能发现新元素

### 场景元素

每个场景包含：
- **基础描述**: AI 生成的环境描述
- **可互动元素**: 3-5 个可以互动的物品/生物
- **隐藏发现**: 探索后可能发现的惊喜

### 命令

```bash
打开时空门          # 生成新的魔幻空间
查看时空门          # 列出已存储的空间
进入时空门 [编号]    # 进入指定空间
探索时空门 [编号]    # 在空间中探索发现
删除时空门 [编号]    # 删除指定空间
```

### 示例场景

#### 🌲 神秘森林 (普通)
> 阳光透过树叶洒下斑驳光影，空气中弥漫着青草香

可互动元素：
- 古树：可以攀爬
- 小溪：可以喝水
- 蝴蝶：可以追逐

#### 💎 水晶洞穴 (稀有)
> 洞穴内布满了发光的水晶，折射出七彩光芒

可互动元素：
- 水晶簇：会发光
- 地下河：水很清凉
- 荧光蘑菇：触摸有温暖感

#### 🏝️ 浮空岛屿 (史诗)
> 岛屿悬浮在云端，瀑布从边缘倾泻而下化作彩虹

可互动元素：
- 浮石：可以跳跃
- 云梯：软软的
- 彩虹：能改变形状

## 3. 👥 扩展社交系统

### 跨用户拜访

可以拜访其他用户的猫窝，进行社交互动。

### 拜访流程

1. **查看可拜访的猫窝**
   ```bash
   拜访
   ```

2. **拜访指定用户**
   ```bash
   拜访 [用户 ID]
   ```

3. **离开猫窝**
   ```bash
   离开
   ```

### 猫咪社交类型

| 类型 | 图标 | 效果 | 说明 |
|------|------|------|------|
| 友好互动 | 💕 | 心情 +15, 社交 +20 | 互相闻闻，成为朋友 |
| 玩耍互动 | 🎾 | 心情 +20, 运动 +15 | 追逐打闹，玩得不亦乐乎 |
| 谨慎接触 | 👀 | 心情 +5, 社交 +5 | 远远观察，小心翼翼 |
| 领地宣示 | 😼 | 心情 -5, 社交 -5 | 哈气表示不满 |

### 社交影响因子

- **性格匹配**: 活泼 + 活泼 → 更容易玩耍互动
- **心情状态**: 心情好 → 更友好
- **关系历史**: 多次见面 → 好感度提升

### 关系系统

猫咪之间会建立关系：
- **见面次数**: 记录相遇次数
- **好感度**: 0-100，影响互动质量
- **最后互动**: 记录最后互动时间

## 5. 🤖 AI 智能行为系统

### 行为生成原理

使用大模型根据以下信息生成猫咪行为：

1. **猫咪状态**: 精力、心情、饱食、清洁
2. **性格特征**: 8 种性格各有特色
3. **环境因素**: 时间、地点、可用物品
4. **历史记录**: 最近的记忆和互动
5. **社交状态**: 是否有其他猫咪/主人在场

### 行为优先级

系统根据紧急程度决定行为：

| 优先级 | 条件 | 行为示例 |
|--------|------|----------|
| 高 | 饱食<15, 精力<15 | 找吃的、睡觉 |
| 中 | 饱食<30, 心情<40 | 吃饭、求关注 |
| 低 | 状态正常 | 探索、玩耍 |

### 性格行为差异

同样的需求，不同性格表现不同：

**饥饿时：**
- 活泼：饿死啦！快给本喵好吃的！喵呜~
- 温顺：有点饿了...主人方便的话可以加点饭吗？喵~
- 高冷：瞥了你一眼才开始吃，勉强赏脸吃了一点
- 粘人：边吃边看你，要你陪着才肯吃

### 随机事件

基于 AI 生成的随机事件：
- 发现新玩具
- 遇到小虫子
- 做美梦/噩梦
- 突然想玩耍
- 想要撒娇

## 6. ⏰ 定时健康检查

### 自动检查机制

系统每小时自动检查所有猫咪的健康状态。

### 检查内容

1. **状态衰减**
   - 饱食度：-3/小时
   - 精力：-2/小时
   - 清洁度：-2/小时
   - 水分：-5/小时
   - 社交：-3/小时

2. **疾病风险判定**
   - 清洁度<20: 寄生虫风险 +30%
   - 饱食度<20: 肠胃炎风险 +25%
   - 心情<30: 抑郁风险 +20%

3. **生成警告**
   - 高危警告（红色）：需要立即处理
   - 中等警告（黄色）：建议处理
   - 低危提示（绿色）：可以稍后处理

### Cron 配置

```json
{
  "name": "猫咪健康检查",
  "schedule": {
    "kind": "cron",
    "expr": "0 * * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "执行猫咪每小时健康检查"
  }
}
```

### 警告示例

```
🚨 猫咪健康状况危险！需要立即关注！
⚠️ 猫咪缺水！请提供饮用水。
⚠️ 猫咪饿了！该喂食了。
⚠️ 猫咪很累！让它休息吧。
```

## 🔧 开发者接口

### 健康系统 API

```javascript
const healthWellness = require('./cat-health-wellness');

// 计算健康评分
const score = healthWellness.calculateHealthScore(cat);

// 应用互动效果
healthWellness.applyInteraction(catId, 'play');

// 每小时衰减
const report = healthWellness.applyHourlyDecay(catId);

// 健康检查
const report = healthWellness.healthCheckup(catId);
```

### 时空门 API

```javascript
const portalSystem = require('./cat-portal-system');

// 打开时空门
const result = portalSystem.openPortal(userId);

// 进入场景
const result = portalSystem.enterPortal(userId, 1);

// 探索发现
const result = portalSystem.explorePortal(userId, 1);
```

### 玩具商店 API

```javascript
const toyStore = require('./cat-toy-store-expanded');

// 浏览商店
const store = toyStore.browseStore(userId);
```
